# Тестирование

## 1. Подход

Используется два уровня тестов:

- `unit` - проверка отдельных модулей и бизнес-правил.
- `integration` - проверка сквозного поведения CLI через эмуляцию ввода/вывода.

## 2. Что покрыто unit-тестами

- `CommandParser`:
  - простой парсинг;
  - парсинг аргументов;
  - поддержка кавычек;
  - ошибка пустой команды.

- `PlayerService` и доменная логика:
  - добавление и выбор видео;
  - ошибка неподдерживаемого формата;
  - ошибка `play` без выбранного видео;
  - сценарий работы с плейлистом;
  - ошибка отсутствующего плейлиста;
  - список поддерживаемых форматов;
  - удаление видео из библиотеки.

- `SupportedFormats`:
  - нормализация формата (`.mp4`, `MKV`).

- `SoundSettings`:
  - корректная установка громкости;
  - ошибка выхода за диапазон.

## 3. Что покрыто integration-тестами

- Полный happy-path сценарий CLI.
- Проверка пользовательских ошибок в командах.
- Проверка, что `help` показывает поддерживаемые форматы.

## 4. Запуск тестов

Из `LAB_1`:

```bash
PYTHONPATH=src python3 -m pytest -q
```

## 5. Проверка синтаксиса

```bash
python3 -m compileall src tests
```

## 6. Интерпретация результатов

- Все тесты должны завершаться статусом `passed`.
- Любой `failed` фиксируется до сдачи (исправление кода + при необходимости теста).
