# Этапы выполнения проекта

## 1. Анализ предметной области

- Выделены сущности: видеоплеер, видеофайл, плейлист, настройки, воспроизведение.
- Зафиксированы бизнес-правила и ограничения.

## 2. Проектирование архитектуры

- Выбрана слоистая структура: `domain` -> `application` -> `cli`.
- Определены ответственности модулей и границы зависимостей.

## 3. Реализация доменной модели

- Реализованы сущности и инварианты.
- Добавлены доменные исключения для нарушений правил.

## 4. Реализация CLI

- Добавлен ручной парсер команд на `shlex`.
- Реализованы роутер и обработчики команд.
- Организована обработка ошибок без аварийного завершения приложения.

## 5. Тестирование

- Добавлены unit-тесты для домена и парсера.
- Добавлены integration-тесты для CLI-сценариев.

## 6. Документация и стабилизация

- Подготовлены документы по архитектуре, командам и тестированию.
