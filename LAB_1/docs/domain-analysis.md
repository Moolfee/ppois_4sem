# Объектно-ориентированный анализ предметной области

## 1. Описание предметной области

Предметная область: управление мультимедийным контентом в видеопроигрывателе.

В рамках учебной задачи мы моделируем систему состояний и команд. Фактического чтения, декодирования и воспроизведения медиафайлов нет.

## 2. Сущности варианта и их отражение в коде

- Видеопроигрыватель -> `VideoPlayer`
- Видеофайл -> `VideoFile`
- Плейлист -> `Playlist`
- Звук -> `SoundSettings`
- Поддерживаемые форматы -> `SupportedFormats`
- Контроль воспроизведения -> `PlaybackControl`

## 3. Атрибуты и ограничения

### `VideoFile`

- `title: str`
- `format_ext: str`
- `duration_seconds: int`

Ограничения:

- длительность должна быть положительным числом;
- формат должен поддерживаться системой при добавлении.

### `SoundSettings`

- `volume: int`, диапазон `0..100`.

### `DisplaySettings`

- `brightness: int`, диапазон `0..100`.

### `PlaybackControl`

- `status`: одно из `stopped`, `playing`, `paused`.

## 4. Бизнес-правила

1. Нельзя добавить видео в неподдерживаемом формате.
2. Нельзя добавить видео с уже существующим названием.
3. Нельзя выполнить `play`, если текущее видео не выбрано.
4. Нельзя задать громкость/яркость вне диапазона `0..100`.
5. Операции с плейлистом разрешены только для существующего плейлиста.
6. Операции с видео в плейлисте разрешены только для существующего видео.
7. При удалении видео из библиотеки оно удаляется из всех плейлистов; если это текущее видео, воспроизведение останавливается.

## 5. Основные сценарии

### Сценарий A: добавить и запустить видео

1. Пользователь добавляет видео в библиотеку.
2. Выбирает видео как текущее.
3. Выполняет `play`.
4. Система переводит состояние в `playing`.

### Сценарий B: управление плейлистом

1. Создать плейлист.
2. Добавить видео в плейлист.
3. Просмотреть содержимое плейлиста.
4. Выбрать видео из плейлиста.

### Сценарий C: удаление видео

1. Удалить видео из библиотеки.
2. Система автоматически удаляет это видео из всех плейлистов.
3. Если удаленное видео было текущим, система делает `stop`.

## 6. Исключения как часть модели

Используются специализированные исключения, которые явно отражают причину ошибки:

- `UnsupportedFormatError`
- `DuplicateVideoError`
- `VideoNotFoundError`
- `PlaylistNotFoundError`
- `PlaylistAlreadyExistsError`
- `PlaybackError`
- `InvalidVolumeError`
- `InvalidBrightnessError`
