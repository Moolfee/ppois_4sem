# LAB_1 - Модель видеопроигрывателя (CLI)

Учебный проект по дисциплине: объектно-ориентированная программная система на Python с интерфейсом командной строки.

Проект не воспроизводит реальные видеофайлы. Он моделирует состояние и поведение видеопроигрывателя: библиотеку видео, плейлисты, операции воспроизведения, настройки громкости и яркости, проверку поддерживаемых форматов.

## Цель проекта

- Показать применение ООП в Python на предметной области "видеопроигрыватель".
- Реализовать ручной CLI без `argparse` и внешних CLI-фреймворков.
- Отработать обработку ошибок через собственные исключения.
- Покрыть ключевую логику unit/integration тестами.

## Функциональность

- Добавление видео в библиотеку.
- Просмотр и удаление видео из библиотеки.
- Выбор текущего видео.
- Управление воспроизведением: `play`, `pause`, `stop`.
- Управление параметрами: `volume`, `brightness`.
- Управление плейлистами: создание, просмотр, добавление/удаление видео, выбор видео из плейлиста.
- Просмотр поддерживаемых форматов.
- Вывод текущего состояния плеера (`status`).

## Архитектура

Проект разделен на 3 слоя:

- `domain` - сущности и бизнес-правила.
- `application` - сценарии использования (сервисный слой).
- `cli` - ручной интерфейс командной строки (парсер, роутер, обработчики).

Подробно: `docs/architecture.md`.

## Структура проекта

```text
LAB_1/

    application/    # use-case сервис
```

## Требования к окружению

- Python 3.11+ (рекомендовано).

## Запуск приложения

Из папки `LAB_1`:

```bash
PYTHONPATH=src python3 -m video_player_cli.main
```

## Запуск тестов

```bash
PYTHONPATH=src python3 -m pytest -q
```

## Проверка "компиляции" (синтаксиса)

```bash
python3 -m compileall src tests
```

## Быстрый пример CLI-сценария

```text
help
formats list
video add "Demo Clip" mp4 90
video list
video select "Demo Clip"
play
volume set 80
brightness set 60
playlist create favorites
playlist add favorites "Demo Clip"
status
video remove "Demo Clip"
status
exit
```

## Ключевые ограничения модели

- Нет реального декодирования или воспроизведения медиа.
- Нет GUI.
- Хранение состояния только в памяти текущего процесса.
